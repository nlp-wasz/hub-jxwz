# 作业1:阅读 gpt4rec的论文，总结实施过程。

1. 问题建模
   将推荐问题重新定义为序列生成任务
   用户的历史交互行为被建模为一个序列
   目标是生成用户未来可能交互的项目
2. 架构设计
   使用GPT的解码器架构作为基础
   采用自回归方式建模用户行为序列
   利用自注意力机制捕获项目间的依赖关系
3. 训练策略
   自监督预训练：在大规模用户行为数据上进行预训练
   微调：在特定推荐任务上进行微调
   采用因果掩码（causal masking）确保预测时不会看到未来信息
4. 序列处理
   将用户的历史交互转换为序列：[item₁, item₂, ..., itemₙ]
   使用位置编码表示项目在序列中的位置
   采用Next-item预测策略：给定前缀序列预测下一个项目
5. 生成式推荐
   与传统方法不同，GPT4Rec采用生成式方法
   可以生成更长的序列，而不仅仅是下一个项目
   支持多样化的推荐结果生成
6. 技术创新
   避免了传统推荐系统中的ID对齐问题
   通过大规模预训练捕获通用的用户行为模式
   具备零样本和少样本学习能力
7. 实施优势
   能够处理可变长度的用户行为序列
   通过预训练获得丰富的语义表示
   生成式方法提供更自然的推荐体验
