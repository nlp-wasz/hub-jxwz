-  数据预处理 ：选取含物品标题的用户交互数据集，剔除噪声数据（如超长标题），对用户交互序列去重、截断，按 8:1:1 划分训练 / 验证 / 测试集，测试集最后一个物品作为预测目标。
-  模型组件配置：采用 GPT-2 作为生成式语言模型，BM25 作为检索引擎；通过提示词格式化用户历史物品标题，输入 GPT-2 学习语言空间下的用户与物品表征。
-  多查询生成：基于束搜索技术，生成多个覆盖用户多维度兴趣的查询词，每个查询精准反映不同兴趣角度。
-  两阶段训练：先以 “用户历史标题 + 目标物品标题” 为训练数据微调 GPT-2；再通过网格搜索优化 BM25 的 K1 和 b 参数
-  物品检索与结果融合：将生成的查询词输入 BM25，按查询评分从每个查询结果中选取对应数量的非重复物品，拼接为最终推荐列表。
-  推荐生成：输入目标用户历史交互数据，经查询生成、检索融合后，输出指定长度的个性化推荐结果。
